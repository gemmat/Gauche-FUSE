1. fuse

Module: fuse

This is a Gauche extension module to use FUSE. This enable you to develop your own filesystem in the Scheme language, without learning kernel module programming.

Callback functions correspond to the system call can tell FUSE its failure by two methods, to raise the <system-error> or to return a negative exact integer which inverted the errno.

To represent errno, the following constants are defined. Each constant is bound to an exact integer representing the system's error number. Note that the actual value may differ among systems, and some of these constants may not be defined on some systems.

Function: getattr path::<string> -> <sys-stat>

Returns a <sys-stat> object as the file infomation of path. If path is a symbolic link, returns a stat of the file the link points.

Function: readlink path::<string> -> <string>

If a file specified by path is a symbolic link, its content is returned. If path doesn't exist or is not a symbolic link, signal an error.

Function: mknod path::<string> mode::<int> rdev::<int> -> <void>

Create the filesystem node(file, device special file or named fifo) named path with the mode mode and the device number rdev.

Function: mkdir pathname::<string> mode::<int> -> <void>

Makes a directory pathname with mode mode.

Function: unlink pathname::<string> -> <void>

Removes pathname.

Function: rmdir pathname::<string> -> <void>

Removes a directory pathname. The directory must be empty.

Function: symlink existing::<string> new::<string> -> <void>

Creates a symbolic link named new to the pathname existing.

Function: rename old::<string> new::<string> -> <void>

Renames a file old to new.

Function: link existing::<string> new::<string> -> <void>

Creates a hard link named new to the existing file existing.

Function: chmod path::<string> mode::<int> -> <void>

Change the mode of the file named path to mode. mode is a small positive integer whose lower 9 bits specifies POSIX style permission.

Function: chown path::<string> owner-id::<int> group-id::<int> -> <void>

Change the owner and/or group of the file named path to owner-id and group-id respectively. owner-id and group-id are an exact integer.

Function: truncate path::<string> length::<int> -> <void>

Truncates a regular file named by path to a size of length bytes. If the file is larger than length bytes, the extra data is discarded. If the file is smaller than that, zero is padded.

Function: open path::<string> file-info::<fuse-file-info> -> <void>

Create a file named by path.

Function: read path::<string> outp::<output port> size::<int> offset::<int> file-info::<fuse-file-info> -> <int>

Output the content of the file named path to outp output port with size bytes length and offset bytes offset. Return the byte length you readed as the file content.

Function: write path::<string> inp::<input port> size::<int> offset::<int> file-info::<fuse-file-info> -> <int>

Write the content of inp input port to the file named path with size bytes length and offset bytes offset. Return the byte length you wrote as the file content.

Function: statfs path::<string> -> <sys-statvfs>

Returns a <sys-statvfs> object as the information about a mounted file system. path is the pathname of any file within the mounted filesystem.

Function: flush path::<string> file-info::<fuse-file-info> -> <void>

Flush the write-buffer of the file named path. This method is optional and can safely be left unimplemented.

Function: release path::<string> file-info::<fuse-file-info> -> <void>

Release the file named path. This method is optional and can safely be left unimplemented.

Function: fsync path::<string> datasync?::<boolean> file-info::<fuse-file-info> -> <void>

Fsync the file named path. This method is optional and can safely be left unimplemented.

Function: setxattr path::<string> name::<string> value::<string> flags::<int> -> <void>

Set the extended file attributes of the file named path with name、value、flags.

Function: getxattr path::<string> name::<string> -> <string>

Return the value of the extended file attributes at name in the string, of the file named path.

Function: listxattr path::<string> -> <string list>

Return the value of the extended file attributes in the string list, of the file named path.

Function: removexattr path::<string> name::<string> -> <void>

Remove the value of the extended file attributes at name, of the file named path.

Function: opendir path::<string> file-info::<fuse-file-info> -> <void>

open the directory corresponding the directory path.

Function: readdir path::<string> file-info::<fuse-file-info> -> <string list>

Returns a list of strings of the directory entries corresponding to the path. The returned list is not sorted. An error is signalled if path doesn't exists or is not a directory.

Function: releasedir path::<string> file-info::<fuse-file-info> -> <void>

Release the directory corresponding to path. Fsync the directory corresponding to path.

Function: init <void> -> <void>

A callback function when the filesystem is initted.

Function: destroy <void> -> <void>

A callback function when the filesystem is destroyed.

Function: access path::<string> mask::<int> -> <boolean>

Returns a boolean value of indicating whether access of path is allowed in mask. This procedure signals an error Create a file named by path with mode mode.

Function: ftruncate path::<string> size::<int> file-info::<fuse-file-info> -> <void>

Same to truncate, except, file-info.

Function: fgetattr path::<string> file-info::<fuse-file-info> -> <sys-stat>

Same to getattr, except, file-info.

Function: utimens path::<string> atime::<int> mtime::<int> -> <void>

Change the file's access time and modification time to atime and mtime, respectively.

Function: start-fuse mountpoint::<string> &keyword args multithreaded::<boolean> getattr readlink mknod mkdir unlink rmdir symlink rename link chmod chown truncate open read write statfs flush release fsync setxattr getxattr listxattr removexattr opendir readdir releasedir fsyncdir init destroy access create ftruncate fgetattr utimens

mountpoint is the path to mount the filesystem. ex. “/tmp/fs/”

args is the FUSE's boot parameter. Provide a parsed argv in the string list format. Default is '(“” “-f” “-s” “-d” mountpoint).

multithreaded is a boolean value to enable multithread (#t) or disable it (#f). Default is #f.

For each rest keyword arguments, provide your callback functions coresponds to the system calls.

Builtin Class: <sys-statvfs>

An object that represents struct statvfs, information of the mounted the filesystem. It has the following slots.

Instance Variable of <sys-statvfs>: f_bsize

file system block size

Instance Variable of <sys-statvfs>: f_frsize

fragment size

Instance Variable of <sys-statvfs>: f_blocks

size of fs in f_frsize units

Instance Variable of <sys-statvfs>: f_bfree

# free blocks

Instance Variable of <sys-statvfs>: f_bavail

# free blocks for non-root

Instance Variable of <sys-statvfs>: f_files

# inodes

Instance Variable of <sys-statvfs>: f_ffree

# free inodes

Instance Variable of <sys-statvfs>: f_favail

# free inodes for non-root

Instance Variable of <sys-statvfs>: f_fsid

file system ID

Instance Variable of <sys-statvfs>: f_flag

mount flags

Instance Variable of <sys-statvfs>: f_namemax

maximum filename length
