1. fuse

Module: fuse

このモジュールは、GaucheからFUSE(Filesystem in Userspace)を使うための拡張ライブラリです。カーネル・モジュール・プログラミングを学ぶ必要なしに、フル機能のファイルシステムをScheme言語で書くことができます。

システムコールに対するコールバック関数は、<system-error>例外を投げるか、errnoの符号を負にした正確整数値を関数の返り値とすることで、システムコールの失敗をFUSEに伝えることができます。

errnoを表現するのに、以下の定義済み定数が使えます。各定数はシス テムエラーを表現する非負正確整数に束縛されています。実際の値はシステム ごとに違い、またシステムによっては定義されていない定数があるということ に注意してください。

Function: getattr path::<string> -> <sys-stat>

与えられたパス名pathで示されるファイルの情報を<sys-stat>オブジェクトで返します。 pathがシンボリックリンクであった場合はリンクの指す先のファイルに関する情報を返します。

Function: readlink path::<string> -> <string>

pathで示されるファイルがシンボリックリンクならば、それが指すパスを返します。 もしpathが存在しなかったり、シンボリックリンクでなければ、エラーを通知します。

Function: mknod path::<string> mode::<int> rdev::<int> -> <void>

pathという名前のファイルシステム・ノード(ファイル、デバイススペシャルファイル、名前付きパイプ)を、属性modeとデバイス番号rdevで作成します。

Function: mkdir pathname::<string> mode::<int> -> <void>

ディレクトリpathnameをモードmodeで作成します。

Function: unlink pathname::<string> -> <void>

pathnameで示されるファイルを消去します。

Function: rmdir pathname::<string> -> <void>

ディレクトリpathnameを消去します。ディレクトリは空でなければなりません。

Function: symlink existing::<string> new::<string> -> <void>

existingを指すnewという名のシンボリックを作成します。

Function: rename old::<string> new::<string> -> <void>

ファイルoldをnewにリネームします。

Function: link existing::<string> new::<string> -> <void>

既存のファイルexistingに対し、newという名のハードリンクを作成します。

Function: chmod path::<string> mode::<int> -> <void>

名前pathを持つファイルのパーミッションビットをmodeに変更します。 modeは小さな正の正確な整数で、POSIXスタイルのパーミッションビットマスク です。

Function: chown path::<string> owner-id::<int> group-id::<int> -> <void>

ファイルpathのオーナーとグループをowner-idとgruop-id で示されるものに変更します。owner-idとgruop-idは正確な整数です。

Function: truncate path::<string> length::<int> -> <void>

pathによって指定される通常ファイルの長さをlengthにします。 ファイルがlengthより長かった場合、余分なデータは捨てられます。 ファイルがlengthより短かった場合、残りの部分にはゼロが詰められます。

Function: open path::<string> file-info::<fuse-file-info> -> <void>

pathという名前のファイルを作成します。

Function: read path::<string> outp::<output port> size::<int> offset::<int> file-info::<fuse-file-info> -> <int>

pathという名前のファイルの内容を、outp出力ポートに、sizeバイトの長さで、offsetのオフセットで、出力してください。 ファイルの内容として読み込んだバイトの長さを返してください。

Function: write path::<string> inp::<input port> size::<int> offset::<int> file-info::<fuse-file-info> -> <int>

pathという名前のファイルへ、inp入力ポートの内容を、sizeバイトの長さで、offsetのオフセットで、書き込んでください。 ファイルの内容として書き込んだバイトの長さを返してください。

Function: statfs path::<string> -> <sys-statvfs>

pathという名前のファイルがある、マウントされたファイルシステムについての情報を、 <sys-statvfs>オブジェクトで返します。

Function: flush path::<string> file-info::<fuse-file-info> -> <void>

pathという名前のファイルへの出力バッファをフラッシュします。 これはオプショナルで、実装しなくても大丈夫です。

Function: release path::<string> file-info::<fuse-file-info> -> <void>

pathという名前のファイルをリリースします。 これはオプショナルで、実装しなくても大丈夫です。

Function: fsync path::<string> datasync?::<boolean> file-info::<fuse-file-info> -> <void>

pathという名前のファイルをfsyncします。 これはオプショナルで、実装しなくても大丈夫です。

Function: setxattr path::<string> name::<string> value::<string> flags::<int> -> <void>

pathという名前のファイルの拡張属性を、name、value、flagsにセットします。

Function: getxattr path::<string> name::<string> -> <string>

pathという名前のファイルの拡張属性のnameの値を、文字列で返します。

Function: listxattr path::<string> -> <string list>

pathという名前のファイルの拡張属性を、文字列のリストで返します。

Function: removexattr path::<string> name::<string> -> <void>

pathという名前のファイルの拡張属性のnameの値を、削除します。

Function: opendir path::<string> file-info::<fuse-file-info> -> <void>

pathに対応するディレクトリを開きます。

Function: readdir path::<string> file-info::<fuse-file-info> -> <string list>

pathに対応するディレクトリの全エントリを文字列のリストとして返します。 リストはソートされません。pathが存在しなかったり、ディレクトリでなかった場合は エラーとなります。

Function: releasedir path::<string> file-info::<fuse-file-info> -> <void>

pathに対応するディレクトリをリリースします。

Function: fsyncdir path::<string> datasync?::<boolean> file-info::<fuse-file-info> -> <void>

pathに対応するディレクトリをfsyncします。

Function: init <void> -> <void>

ファイルシステムが初期化されるときのコールバックです。

Function: destroy <void> -> <void>

ファイルシステムが破棄されるときのコールバックです。

Function: access path::<string> mask::<int> -> <boolean>

pathへのアクセスが maskに示されるモードで許可されているかどうかを示す真偽値を返します。

Function: create path::<string> mode::<octat int> file-info::<fuse-file-info> -> <void>

pathという名前のファイルをモードmodeで作成します。

Function: ftruncate path::<string> size::<int> file-info::<fuse-file-info> -> <void>

truncateと同じですが、file-infoがあります。

Function: fgetattr path::<string> file-info::<fuse-file-info> -> <sys-stat>

getattrと同じですが、file-infoがあります。

Function: utimens path::<string> atime::<int> mtime::<int> -> <void>

ファイルのアクセスタイムと変更タイムをatimeとmtimeが示す値にセットします。

Function: start-fuse mountpoint::<string> &keyword args multithreaded::<boolean> getattr readlink mknod mkdir unlink rmdir symlink rename link chmod chown truncate open read write statfs flush release fsync setxattr getxattr listxattr removexattr opendir readdir releasedir fsyncdir init destroy access create ftruncate fgetattr utimens

mountpointは、ファイルシステムをマウントするパスです。(“/tmp/fs/”など)

argsは、FUSEの起動オプションです。argvをパースしたものを、文字列のリストで渡してください。(デフォルトは'(“” “-f” “-s” “-d” mountpoint))

multithreadedは、真偽値で、マルチスレッドを有効(#t)、無効(#f)にします。デフォルトは#fです。

残りのキーワード引数にはそれぞれ、システムコールに対応するコールバック関数を与えてください。

Builtin Class: <sys-statvfs>

マウントされたファイルシステムの情報を表す、struct statvfsのラッパー オブジェクトです。以下に示すスロットを持ちます。

Instance Variable of <sys-statvfs>: f_bsize

ファイルシステムのブロックサイズ

Instance Variable of <sys-statvfs>: f_frsize

フラグメントサイズ

Instance Variable of <sys-statvfs>: f_blocks

ファイルシステムのサイズ (f_frsize 単位)

Instance Variable of <sys-statvfs>: f_bfree

解放されているブロック数

Instance Variable of <sys-statvfs>: f_bavail

ルート以外の解放されているブロック数

Instance Variable of <sys-statvfs>: f_files

inode 数

Instance Variable of <sys-statvfs>: f_ffree

解放されている inode の数

Instance Variable of <sys-statvfs>: f_favail

ルート以外の解放されている inode の数

Instance Variable of <sys-statvfs>: f_fsid

ファイルシステム ID

Instance Variable of <sys-statvfs>: f_flag

マウントフラグ

Instance Variable of <sys-statvfs>: f_namemax

ファイル名の長さの最大値
